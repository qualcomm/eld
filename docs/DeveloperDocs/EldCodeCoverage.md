# ELD Code Coverage Guide

## Overview

LCOV is a command-line tool that captures, merges, filters, and summarises coverage data. For Clang-based projects like ELD, we recommend using lcov with a wrapper script that invokes llvm-cov by passing gcov as an argument to ensure compatibility with Clang-generated .gcno and .gcda files.

The lcov pipeline relies on two types of files generated by the compiler:

- **`.gcno` (Graph Notes)** — Generated at **compile time**. Contains the control flow graph of the program used to map coverage data back to source lines.
- **`.gcda` (Graph Data)** — Generated at **runtime** when instrumented binary is executed. Contains the actual hit counts for lines, functions, and branches.

---

## Prerequisites

Before starting, ensure lcov is available. If not, install it:
- Install on Debian/Ubuntu: `sudo apt-get install lcov`
- genhtml comes bundled with the lcov package.

---

## Environment Setup

Export the following environment variables:

```bash
export CC=clang
export CXX=clang++
```

## Code Coverage Build

Configure the build with code coverage enabled:

```bash
cmake -G Ninja \
  -DCMAKE_BUILD_TYPE=Debug \
  -DCMAKE_INSTALL_PREFIX=../inst \
  -DLLVM_ENABLE_ASSERTIONS:BOOL=ON \
  -DLLVM_ENABLE_PROJECTS='llvm;clang' \
  -DLLVM_EXTERNAL_PROJECTS=eld \
  -DLLVM_EXTERNAL_ELD_SOURCE_DIR=${PWD}/llvm-project/eld \
  -DLLVM_TARGETS_TO_BUILD='ARM;AArch64;RISCV;Hexagon;X86' \
  -DCMAKE_CXX_FLAGS='-stdlib=libc++' \
  -DLLVM_USE_LINKER=lld \
  -DELD_COVERAGE=ON \
  -B ./obj \
  -S ./llvm-project/llvm
```

Once configured, build the project.

```bash
cmake --build obj
```

## Generating Code Coverage Report

### Step 1 — Setup llvm-gcov.sh Wrapper

Before running the lcov pipeline, we need the wrapper `llvm-gcov.sh`. Its task is to point lcov to `llvm-cov` instead of the system gcov, which is incompatible with Clang-generated `.gcno` files.

Placing `llvm-gcov.sh` inside the build directory and add it to PATH so that all lcov commands can be run consistently from the project root.

Create the wrapper inside `obj/`:

```bash
cat > obj/llvm-gcov.sh << 'EOF'
#!/bin/bash
exec llvm-cov gcov "$@"
EOF
```

Make it executable:

```bash
chmod +x obj/llvm-gcov.sh
```

Add the wrapper directory to PATH:

```bash
export PATH=$PWD/obj:$PATH
```


### Step 2 — Reset Counters

Reset coverage counters by removing stale coverage data files (.da/.gcda) from previous runs. This step is safe to run even on a fresh build:

```bash
lcov --zerocounters --directory obj \
  --rc lcov_branch_coverage=1
```

### Step 3 — Capture Baseline (Before Tests)

Capture the initial coverage state before running any tests. This reads the compiled `.gcno` files and records all source lines as zero-hit, ensuring that unexecuted code is correctly reflected in the final report:

```bash
lcov --capture --initial \
  --directory obj \
  --gcov-tool llvm-gcov.sh \
  --output-file cov_base.info \
  --rc lcov_branch_coverage=1
```

### Step 4 — Run Tests

Run the full ELD test suite to generate runtime coverage data:

```bash
cmake --build obj -- check-eld
```


### Step 5 — Capture Results (After Tests)

Capture the coverage data generated during the test run. This reads the `.gcda` files produced at runtime and records all lines, functions, and branches that were executed:

```bash
lcov --capture \
  --directory obj \
  --gcov-tool llvm-gcov.sh \
  --output-file cov_test.info \
  --rc lcov_branch_coverage=1
```

### Step 6 — Merge Both Info Files

After capturing both baseline and test data, merge `cov_base.info` and `cov_test.info` to produce a complete coverage data file that includes zero-hit lines alongside executed ones:

```bash
lcov -a cov_base.info -a cov_test.info \
  -o cov.info \
  --rc lcov_branch_coverage=1
```

### Step 7 — Extract ELD Sources Only

Since ELD is built alongside LLVM and Clang, `cov.info` contains coverage entries for LLVM/Clang source files as well. Use `--extract` to isolate only ELD source files:

```bash
lcov --extract cov.info '*/eld/*' \
  --output-file cov_eld.info \
  --rc lcov_branch_coverage=1
```

### Step 8 — Remove Non-Essential Paths

Before generating the HTML report, remove non-essential entries from `cov_eld.info`.

```bash
lcov -r cov_eld.info \
  '*/test/*' \
  '*/Plugins/HelloWorldPlugin/*' \
  --output-file cov_eld_filtered.info \
  --rc lcov_branch_coverage=1
```

### Step 9 — Generate HTML Report

Generate an annotated HTML report with line, function, and branch coverage from `cov_eld_filtered.info`:

```bash
genhtml --branch-coverage \
  --function-coverage \
  --ignore-error source \
  -o coverage-report \
  cov_eld_filtered.info \
  --rc genhtml_branch_coverage=1
```

The HTML report will be generated under the `coverage-report` directory. Open `coverage-report/index.html` in a browser to view the report.

### Step 10 — Quick Coverage Summary

To quickly extract the final coverage percentages from cov_eld_filtered.info, use the following commands:

**Line Coverage:**
```bash
lcov --summary cov_eld_filtered.info 2>&1 | grep "lines" | awk '{print $2}'
```

**Function Coverage:**
```bash
lcov --summary cov_eld_filtered.info 2>&1 | grep "functions" | awk '{print $2}'
```

**Branch Coverage:**
```bash
lcov --summary cov_eld_filtered.info --rc lcov_branch_coverage=1 2>&1 | grep "branches" | awk '{print $2}'
```
